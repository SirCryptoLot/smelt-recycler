Mobile wallet connect — which approach?

3 ways to handle wallet detection on mobile. The core problem: on Android/iOS Chrome, wallet extensions don't exist — you need deep links to open inside a wallet's browser.

A

Custom bottom-sheet modal with auto-detection

Replace the current deep-link buttons with a proper modal. Detect if already inside a wallet browser (Phantom, Backpack, Solflare). If yes → show "Connect" for that wallet. If not → show all wallet deep-link buttons (Phantom, Backpack, Solflare, Jupiter).

Pros

  • Best UX — detects context, shows right option
  • Covers all 4 wallets
  • Clean, branded feel

Cons

  • Most code to write
B

Keep WalletMultiButton + add missing adapters + fix detection

Add Backpack adapter to providers.tsx (already has Phantom + Solflare). Fix isInWalletBrowser() to detect Backpack. Update the existing deep-link drawer buttons to include Backpack and Jupiter links.

Pros

  • Minimal change, least risk
  • Reuses existing WalletMultiButton (standard UI)

Cons

  • WalletMultiButton is generic/unbranded
  • Mobile deep link drawer still basic
C

Wallet-standard auto-discovery + custom connect button

Use @wallet-standard/app to auto-discover any registered wallet (Phantom, Backpack, etc register themselves at page load). Build a small custom "Connect" modal that lists discovered wallets by name, with deep links for the rest on mobile.

Pros

  • Future-proof — detects any wallet-standard wallet automatically
  • Clean custom UI

Cons

  • Requires understanding wallet-standard events
  • Medium complexity

My recommendation: A — gives the best user experience with clear detection logic and covers all 4 wallets cleanly.
