SMELT Token — Evolution Options

Current SMELT is a simple reward token. Here are 3 directions to make it more interesting.

A

Keep current + show live NAV

SMELT stays as-is (emissions reward, manual weekly distributions). Add one powerful display: a live NAV price — pending SOL pool ÷ total supply — so holders can see exactly what each SMELT is worth right now. Admin UI shows it. Pools page shows it. Simple, honest, no new program needed.

✅ No new on-chain code  ·  ✅ Ships this week  ·  Shows value clearly
B

JLP-style yield token (on-chain vault)

SMELT becomes a yield-bearing receipt token like JLP. All platform SOL fees flow into an on-chain vault. NAV = vault SOL ÷ SMELT supply — it rises automatically as fees accumulate. Holders redeem SMELT for their share of SOL at any time. No weekly admin run needed. Price discovery is real and transparent.

⚠️ Needs new Anchor program (~2 SOL deploy)  ·  Most DeFi-native  ·  Like stSOL or JLP
C

Algorithmic emission + deflationary burn

Keep the existing distribution model but add two mechanics: (1) adaptive emission — rate adjusts based on 7-day recycling volume (high activity = lower rate, keeping scarcity), and (2) a burn-on-claim — when SOL is distributed, a small % of SMELT is burned from the pool. Supply shrinks over time. Each remaining token gets rarer and more valuable. Halvings + burns = strong scarcity narrative.

⚠️ Needs staking program update  ·  Strong tokenomics story  ·  Requires more activity data

My recommendation: Start with A now (ships immediately, makes SMELT tangibly valuable to holders today), build toward B once you have users and the SOL for program deploy. Option C is the most ambitious tokenomics but needs real activity data to tune the adaptive emission curve.
